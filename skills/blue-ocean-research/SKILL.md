# 蓝海市场探索

本 skill 依赖 web-access 的联网策略调度能力完成数据采集。执行前必须先加载 web-access skill 并遵循其浏览哲学。

## 平台分工

| 平台 | 用途定位 | 工具 | 采集方式 |
|------|---------|------|---------|
| 小红书 | 内容蓝海探测、用户需求挖掘 | browser | 搜索页 evaluate 提取笔记列表、点击进入详情评论区 |
| 微博 | 舆论热度、话题讨论度 | browser | 搜索页 evaluate 提取微博列表 |
| 1688 | 供应链验证、源头货源 | browser | 搜索页 evaluate 提取商品/供应商信息 |
| 京东 | 电商竞品格局、用户评价 | browser | 搜索页 + 评价页 evaluate 提取 |
| 淘宝/天猫 | 销量排序、价格带分布 | browser | 按销量排序搜索，evaluate 提取列表 |
| 什么值得买 | 比价/优惠热度 | browser | 搜索页 evaluate 提取 |
| 抖音 | 视频内容搜索 + 话题热度 | browser | 搜索页 + 话题页 evaluate |
| 灰猫数据 | 小红书商业化数据（核心） | browser | 商品榜单、达人搜索、直播数据 |
| 亚马逊 | 跨境/全球需求验证 | browser/web_fetch | 搜索页或 Movers & Shakers |

采集前先确认目标网站，检查 web-access 的 `references/site-patterns/` 下是否有匹配站点的经验文件。有先验知识时优先遵循。

## 触发识别

**模式 A（品类/赛道 → 找蓝海）**：用户给方向词，想找机会
- 例：「研究 XX 赛道」「XX 品类有没有蓝海」「XX 最近有什么品在上升」

**模式 B（具体品 → 验证机会）**：用户给品名/品牌/SKU
- 例：「帮我分析 XXX 这个品」「XXX 在小红书数据怎么样」

## 模式 A：品类蓝海研究

### 第一步：拆解品类词

用户给的是大方向时（如"大健康"「美妆"），先拆成 3-5 个具体品类词，向用户确认。

> 例：大健康 → 益生菌、维生素 D3、重组胶原蛋白、睡眠辅助、叶黄素

---

### 第二步：与用户确认平台清单

**操作流程**：
1. 用 `find-url.mjs` 检索用户本地 Chrome 是否有目标平台的登录页访问记录
2. 告知用户本次将采集的平台，默认六件套：小红书 / 灰猫 / 京东 / 淘宝 / 1688 / 抖音；微博 / 什么值得买 / 亚马逊 视场景可选
3. 用户确认后，显式写出最终平台清单再进入下一步

> ⚠️ 不要跳过确认：跳过会导致采集命令冲突、漏采或多采。

---

### 第三步：登录预检

用户确认平台后，对每个平台做最小化探测：

```
1. browser navigate → 打开目标平台首页或搜索页
2. browser evaluate → 检查页面是否已登录（判断标准：能否看到个人信息或能否执行搜索）
3. 未登录的平台，告知用户："请在你的 Chrome 中登录 [XXX]，完成后告诉我继续"
4. 必须等用户回复“继续”再进入下一步
```

**登录判断核心原则**（来自 web-access）：**目标内容拿到了吗？** 打开页面先尝试获取内容，确认目标无法获取且判断登录能解决时才告知用户。

---

### 第四步：多平台内容 + 电商扫描

**采集原则**：
- 平台间命令**必须串行**（同一 Chrome 实例，并发会触发风控/竞态）
- 每次采集先 `browser navigate` 到目标页面，再 `evaluate` 提取数据
- 提取完数据后保留原始记录，分析时引用

#### 4a. 内容平台扫描

**小红书**（最核心）：
```
browser navigate → https://www.xiaohongshu.com/search_result?keyword=<品类词>
browser scroll → 触发懒加载
browser evaluate → 提取笔记列表（标题、点赞数、作者、URL）
```

扩展搜索词：
- `<品类词> 推荐` — 看种草密度
- `<品类词> 避坑` — 看用户痛点
- `<品类词> 测评` — 看内容深度

**评论区挖掘**（核心需求信号）：
```
1. 从搜索结果中选择高互动笔记，browser click 进入详情
2. browser scroll → 滚动到评论区
3. browser evaluate → 提取评论内容（用户、内容、点赞数）
```

**微博**（话题是否出圈）：
```
browser navigate → https://s.weibo.com/weibo?q=<品类词>
browser evaluate → 提取微博列表（内容、转发/评论数、热度）
```

**抖音**（视频内容 + 话题热度）：
```
browser navigate → https://www.douyin.com/search/<品类词>
browser evaluate → 提取视频列表
```

**内容蓝海判断维度**：

| 信号 | 判断 |
|------|------|
| 笔记结果少 + 点赞均值高 | ✅ 蓝海：需求强但供给少 |
| 笔记同质化严重，标题雷同 | ⚠️ 内容红海，需差异化切入 |
| 评论区高频："哪里买"“求推荐"“用了多久" | ✅ 强购买意图 |
| 笔记以"避坑"“踩雷"为主 | 💡 用户对现有品不满，新品机会 |
| 粉丝少的博主也能出爆款 | ✅ 算法在扶持这个方向 |
| 微博有讨论但小红书少 | ⚡ 大众需求存在，垂直内容空白 |

#### 4b. 电商平台扫描

核心四问：**能不能拿货？盘子多大？用户真正不满意什么？内容可以怎么切？**

**① 1688（供应链维度）**：
```
browser navigate → https://s.1688.com/selloffer/offer_search.htm?keywords=<品类词>
browser evaluate → 提取商品数量、价格区间、MOQ信息
```

| 信号 | 判断 | 内容机会 |
|------|------|---------|
| 大量SKU + 价格低 + MOQ低 | 供应链成熟，竞争激烈 | 内容差异化是唯一出路 |
| 有货但MOQ高 | 工厂品，尚未零售化 | 💡 做「怎么挑选「「工厂直供」类内容 |
| SKU极少 / 几乎没有 | 供应链空白 | 需配合电商销量双重验证 |

**② 京东（竞品格局 + 用户痛点）**：
```
browser navigate → https://search.jd.com/Search?keyword=<品类词>
browser evaluate → 提取商品列表（品牌、价格、评价数、榜单标签）
# 选择 1-2 个热销品，进入评价页
browser click → 进入详情页
browser evaluate → 提取差评（1-2星）内容
```

**③ 淘宝（销量排序 + 价格带）**：
```
browser navigate → https://s.taobao.com/search?q=<品类词>&sort=sale-desc
browser evaluate → 提取销量、价格、店铺信息
```

**④ 什么值得买（价格敏感度）**：
```
browser navigate → https://search.smzdm.com/?c=home&s=<品类词>
browser evaluate → 提取热度、价格讨论
```

**⑤ 亚马逊（全球需求验证）**：
```
browser navigate → https://www.amazon.com/s?k=<品类英文>
# 或
web_fetch → https://www.amazon.com/gp/movers-and-shakers
browser evaluate → 提取搜索结果量、Rating、Review数
```

### 第五步：灰猫商业数据验证

对第四步筛选出的**潜力词**，用灰猫验证商业转化：

```
# 商品销量榜
browser navigate → https://xhs.huitun.com/
# 在灰猫搜索框输入品类词，进入榜单页
browser evaluate → 提取销量、GMV、笔记数、直播数、趋势
```

**双向验证逻辑**：

| 社交媒体信号 | 灰猫验证 | 结论 |
|------------|---------|------|
| 内容热度高 | 销量也高 | ✅ 成熟赛道，竞争激烈，需差异化 |
| 内容热度高 | 销量低/无商品 | 🌱 内容蓝海，商业化待开发，先机 |
| 内容热度低 | 销量突然高 | ⚡ 黑马品，内容还没跟上，立即布局 |
| 内容热度低 | 销量也低 | ❌ 市场未成熟，观望 |

```

browser evaluate → 提取笔记列表
# 选择高互动笔记，进入评论区

# 参照模式 A 4b 的采集流程
```

**印证维度**：

| 灰猫数据 | 社媒/电商印证 | 综合结论 |
|---------|-----------|---------|
| GMV高、note多 | 笔记互动高、评论正向 | ✅ 品强 + 内容强，成熟爆品 |
| GMV高、note少 | 搜索结果稀少 | ⚡ 靠直播/投放驱动，内容空白 |
| GMV低、note多 | 笔记互动高但评论多"哪里买" | 🌱 内容热但转化弱，供应链/定价问题 |
| GMV低、note少 | 社媒也少 | ❌ 无信号，跳过 |
| 直播GMV高、笔记少 | 淘宝/京东有对应销量 | ⚡ 直播黑马，笔记内容窗口未打开 |
| 达人以头部为主 | 京东/淘宝差评集中 | 💡 腰部达人有入场机会 |

## 十一、蓝海性质判定（必填，缺失视为不合格）

### 五级判定标尺

| 级别 | 判定条件（必须同时满足） | 含义 |
|------|---------------------|------|
| ✅ 深蓝海 | 内容供给少 + 需求强信号 + 1688货源成熟 + 头部Top3占比<60% | 真空白：需求被验证，供给侧没人占位 |
| 🌱 浅蓝海 | 内容已有但同质化严重 + 用户对现有品有明显不满 + 价格带集中 | 内容侧已挤但产品/角度仍可差异化切入 |
| ⚡ 黑马窗口 | 灰猫销量/直播GMV近期突增但小红书内容稀薄 + 京东/淘宝动销飙升 | 商业化跑在内容前面，内容侧抢占红利的最后窗口期 |

## 执行注意事项

1. **必须先加载 web-access skill**：所有联网操作通过 web-access 的工具选择策略执行，不要自己决定用什么工具。
2. **平台间必须串行**：browser 操作走的是同一 Chrome 实例，并发会触发风控/竞态。多个平台一定要串行采集。
3. **关键词精准**：用具体品类词而非大词（"益生菌"而非"保健品"）。
4. **社媒评论要抓**：高互动笔记的评论区是最真实的用户需求信号。
5. **差评是金矿**：京东 1-2 星差评 = 用户未被满足的需求 = 内容可以解答的问题。
6. **必等用户登录**：未登录平台不要硬跑，告知用户登录后继续。
7. **飞书文档**：先用 lark-shared/SKILL.md 确认登录态，再用 `lark-cli docs +create`。

### 采集优先级

3. **可选验证**（缺失不影响核心判断）：淘宝销量排序 + 1688供应链 + 抖音视频 + 亚马逊跨境
