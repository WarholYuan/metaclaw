---
title: Process Management
description: Manage MetaClaw process lifecycle with metaclaw commands
---

Process management commands control the MetaClaw background process. These commands are only available in the terminal.

## start

Start the MetaClaw service. Runs as a background daemon by default and automatically tails logs.

```bash
metaclaw start
```

**Options:**

| Option | Description |
| --- | --- |
| `-f`, `--foreground` | Run in foreground, not as a background daemon |
| `--no-logs` | Don't tail logs after starting |

## stop

Stop the running MetaClaw service.

```bash
metaclaw stop
```

## restart

Restart the MetaClaw service (stop then start).

```bash
metaclaw restart
```

**Options:**

| Option | Description |
| --- | --- |
| `--no-logs` | Don't tail logs after restart |

## update

Update code and restart the service. Automatically performs:

1. Pull latest code (`git pull`)
2. Stop current service
3. Update Python dependencies
4. Reinstall CLI
5. Start service

```bash
metaclaw update
```

<Warning>
  If `git pull` fails (e.g., uncommitted local changes), the update aborts and the service remains unaffected.
</Warning>

## status

Check MetaClaw service status, including process info, version, and current model/channel configuration.

```bash
metaclaw status
```

## logs

View service logs.

```bash
metaclaw logs
```

**Options:**

| Option | Description | Default |
| --- | --- | --- |
| `-f`, `--follow` | Continuously tail log output | No |
| `-n`, `--lines` | Show last N lines | 50 |

Examples:

```bash
# View last 100 lines
metaclaw logs -n 100

# Continuously tail logs
metaclaw logs -f
```

## install-browser

Install Playwright and Chromium browser for the [browser tool](/en/tools/browser).

```bash
metaclaw install-browser
```

<Tip>
  Only needed when using browser tools (web browsing, screenshots, etc.).
</Tip>

## run.sh Compatibility

If Cow CLI is not installed, you can use `run.sh` to manage the service:

| metaclaw command | run.sh equivalent |
| --- | --- |
| `metaclaw start` | `./run.sh start` |
| `metaclaw stop` | `./run.sh stop` |
| `metaclaw restart` | `./run.sh restart` |
| `metaclaw update` | `./run.sh update` |
| `metaclaw status` | `./run.sh status` |
| `metaclaw logs` | `./run.sh logs` |

<Note>
  The `metaclaw` command is recommended — it provides cleaner syntax and richer features. It is automatically installed via the one-click install script.
</Note>
